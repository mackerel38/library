# DP recipes

`cp/dp/`へ万能APIを作ると状態定義がかえって長くなる解法を、実装時の確認項目として管理する。

- 線形DP: 状態、初期値、遷移方向、到達不能値、復元元を先に決める。
- 桁DP: `position`, `tight`, `started`と問題固有状態を分離する。
- 期待値DP: 自己遷移を右辺から移項してから、依存順を確認する。
- 区間DP: 半開区間`[left, right)`を使い、短い区間から計算する。
- 木DP: 親側へ渡す境界状態と、子の併合順を分離する。
- frontier DP: 同じ連結状態を同一キーに正規化し、消えた成分の可否を判定する。
- 挿入DP: 値そのものではなく相対順位・挿入位置を状態にできないか検討する。
- DP最適化: 単調性やMonge性を証明できないまま最適化ヘッダを使わない。

全66問の具体的な分類は`docs/dp/catalog.md`を参照する。
